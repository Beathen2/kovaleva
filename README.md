Медицинская платформа записи к врачу
Описание проекта
Веб-приложение для онлайн-записи пациентов к врачам с возможностью ведения медицинских заключений и скачивания отчётов.
Система реализует взаимодействие между тремя ролями:
•	Врач
•	Пациент
•	Администратор
Основной функционал
Пациент
•	Регистрация и авторизация
•	Просмотр списка врачей
•	Запись на приём
•	Просмотр своих записей
•	Удаление записей
•	Просмотр заключений врача
•	Скачивание заключения в PDF

Врач
•	Просмотр записей пациентов
•	Ведение заключений
•	Завершение приёма (смена статуса)

Администратор
•	Доступ к панели управления (базовый функционал)

Используемые технологии
Frontend
•	React
•	React Router
•	JavaScript (ES6+)
•	CSS
Backend
•	Node.js
•	Express
•	CORS
Дополнительно
•	jsPDF — генерация PDF-файлов

Архитектура проекта
Приложение построено по клиент-серверной архитектуре:
•	Frontend (React) — пользовательский интерфейс
•	Backend (Express) — обработка запросов и логика
•	Данные хранятся в памяти (без БД)

Установка и запуск
1. Клонирование проекта
git clone https://github.com/Beathen2/kovaleva.git
cd kovaleva
 
2. Установка зависимостей
Backend
npm install express cors
Frontend
npm install

3. Запуск проекта
Запуск сервера
node server.js
Сервер запустится на:
http://localhost:3001
Запуск клиента
npm start

API (Backend)
Авторизация
•	POST /api/login
•	POST /api/register

Врачи
•	GET /api/doctors

Записи
•	POST /api/appointments — создать запись
•	GET /api/appointments/patient/:id — записи пациента
•	GET /api/appointments/doctor/:id — записи врача
•	DELETE /api/appointments/:id — удалить запись

Заключения
•	POST /api/reports — создать заключение
•	GET /api/reports/patient/:id — получить заключения

Логика работы системы
1.	Пользователь регистрируется или входит в систему
2.	Пациент выбирает врача и записывается
3.	Врач видит запись и добавляет заключение
4.	Приём завершается → статус меняется на done
5.	Пациент может скачать заключение в PDF
